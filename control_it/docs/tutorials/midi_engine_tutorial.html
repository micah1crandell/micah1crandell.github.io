Mastering the MIDI Engine

The MIDI Engine allows you to run Control It actions using physical hardware controllers (like a Korg nanoKONTROL or MIDI footswitch), even while the app is in the background.

Concept

Unlike standard MIDI Mapping (which controls the UI), the MIDI Engine is a separate "layer" that listens for incoming MIDI messages and fires actions directly. This means you can trigger an HTTP request or an OSC command without ever touching the screen.

How to Setup

1. Connect Hardware

Connect your MIDI controller via USB (using the camera adapter) or Bluetooth MIDI.

2. Create a Trigger

Go to Settings > Engines > MIDI Engine and tap +.

3. Use "Learn Mode"

Instead of manually typing channels and note numbers:

  1. Tap the Learn button.
  2. Press the physical button or key on your MIDI controller.
  3. The app will automatically detect the Message Type, Channel, and Value.

4. Assign an Action

Choose what happens when this trigger fires. You can select any existing Action from your grids.

Background Audio

Important: To keep the MIDI Engine running when you lock your phone or switch apps, ensure Background Audio mode is enabled in Settings. This plays a silent audio track to keep the operating system from suspending the app.

Advanced: Ranges

Triggers can also work with ranges (e.g., faders). You can set a trigger to fire only when a CC value is within a specific range (e.g., 100-127), allowing you to create "zones" on a single fader.
