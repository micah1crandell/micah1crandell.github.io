Using the HTTP Server

Turn your iPhone or iPad into a web server! This allows external devices, scripts, or apps (like Stream Deck) to trigger actions inside Control It by simply loading a URL.

Enabling the Server

  1. Go to Settings > Engines > HTTP Trigger Server.
  2. Tap Start Server.
  3. Note the Server URL (e.g., http://192.168.1.50:8080).

Creating a Trigger

Instead of a fixed API, you define your own custom endpoints.

  1. Go to the Triggers tab in the Server settings.
  2. Tap + to create a new trigger.
  3. Name: A friendly name (e.g. "Scene 1").
  4. Endpoint: The URL path (e.g. /actions/scene1).
  5. Method: GET (or POST/PUT).
  6. Action: Link it to the grid action you want to fire.

Triggering Actions

Once created, you can trigger the action by sending a request to the full URL:

http://<Your-IP>:8080/actions/scene1

If successful, the server will respond with 200 OK.

Security

You can optionally enable Bearer Token Authentication for each trigger if you need to secure your endpoints.

Example: Stream Deck Integration

You can use a Stream Deck to fire Control It actions.

  1. In the Elgato Stream Deck software, drag in a Website action.
  2. Set the URL to your trigger link (e.g. http://192.168.1.50:8080/actions/scene1).
  3. Check "GET request in background".

Now, pushing the physical button on your Stream Deck will instantly fire the associated action in Control It!
