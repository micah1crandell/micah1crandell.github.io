TCP & UDP Protocols

For raw communication with hardware like matrix switchers, projectors, and industrial controllers, Control It offers direct TCP and UDP socket support.

Protocol Selection

TCP (Transmission Control Protocol)

Reliable, connection-oriented. Guarantees delivery and order. Best for projectors and matrix switchers.

UDP (User Datagram Protocol)

Fast, simpler, connectionless. No guarantee of delivery. Best for "fire and forget" commands or high-speed telemetry.

Terminators

Many hardware devices require a specific "End of Line" (EOL) character to know when a command is finished. Control It appends these automatically based on your selection:

Terminator Hex Description
None -- Sends the raw string exactly as typed.
\r (CR) 0x0D Carriage Return. Common for projectors and older serial devices.
\n (LF) 0x0A Line Feed. Common for Linux/Unix systems.
\r\n (CRLF) 0x0D0A Carriage Return + Line Feed. The standard network terminator (e.g., HTTP).

Connection Pooling (TCP)

Establishing a TCP connection takes time (handshake). To improve performance, Control It uses Connection Pooling.

  • Keep-Alive: The app attempts to keep the connection open for a short period after sending a command.
  • Reuse: If you press a button again within this window, it reuses the existing socket instead of opening a new one.
  • Auto-Reconnect: If the connection drops or times out, the app will automatically attempt to reconnect on the next button press.

Binary Data

You can send binary data (hex codes) by using standard escape sequences in your string:

Use format: \xHH

Example: To send the byte sequence 02 00 00 03, enter: \x02\x00\x00\x03
