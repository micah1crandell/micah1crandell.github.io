Open Sound Control (OSC)

Control It supports the robust OSC protocol for low-latency communication with media servers, audio software, and creative coding environments.

Anatomy of a Message

Address Pattern

The routing path for the message. Must start with `/`.

Example: /layer/1/opacity

Type Tag

A string telling the receiver what data types follow.

Example: ,if (Integer, Float)

Arguments

The actual data payload values.

Example: 1, 0.5

Argument Types

Control It automatically infers the OSC type based on your input format:

Input Format OSC Type Tag Example
Integer Int32 i 42, -100
Decimal Float32 f 0.5, 3.14159
Keyword Boolean T/F true, false
Text String s "Hello", clip_1

Advanced Features

Wildcards

Control It fully supports standard OSC pattern matching:

  • * matches any sequence of characters (e.g., `/layer/*/opacity`)
  • ? matches any single character
  • [1-5] matches a range of numbers
  • {foo,bar} matches a specific list (e.g., `/layer/{1,2}/opacity`)

Auto-Release Logic

For momentary buttons, Control It can automatically send a "release" message (0) shortly after the press message.

Trigger Condition: The action sends an Integer 1 or Boolean true as the first argument.

Release Action: After ~100ms, the app sends the same address with value 0 or false.
