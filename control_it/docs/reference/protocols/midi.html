MIDI Protocol Reference

Control It is a fully compliant CoreMIDI device. It supports sending and receiving MIDI messages over USB, Bluetooth, and Network (RTP-MIDI).

Supported Message Types

All standard voice messages are supported on all 16 channels:

Type Hex Status Data 1 Data 2
Note Off 8x Note No. (0-127) Velocity (0-127)
Note On 9x Note No. (0-127) Velocity (0-127)
Poly Aftertouch Ax Note No. (0-127) Pressure (0-127)
Control Change (CC) Bx Controller (0-127) Value (0-127)
Program Change Cx Program (0-127) --
Channel Aftertouch Dx Pressure (0-127) --
Pitch Bend Ex LSB (0-127) MSB (0-127)

System Messages

Control It can also send global system messages, which are independent of channel.

  • Sysex: Not currently customizable via the UI.
  • MMC (MIDI Machine Control): Supported commands include Play, Stop, Record, Rewind, Fast Forward (7F 7F 06 xx).
  • Real-Time: Start, Stop, Continue, and Clock messages are handled by the MIDI Clock Engine.

RPN and NRPN

Registered (RPN) and Non-Registered Parameter Numbers (NRPN) are complex messages typically used for detailed synth control (like envelope attack or filter resonance) that require 14-bit resolution.

Since RPN/NRPN are actually a sequence of multiple CC messages (CC 98/99/6/38), you must currently construct these using a Multistep Action:

  1. CC 99 (NRPN MSB)
  2. CC 98 (NRPN LSB)
  3. CC 6 (Data Entry MSB)
  4. CC 38 (Data Entry LSB) - Optional for fine control

Network MIDI (RTP-MIDI)

Control It fully supports Apple's RTP-MIDI protocol over WiFi.

  • Latency: Typically 1-4ms on a good 5GHz WiFi network.
  • Jitter: The app timestamps all outgoing packets to ensure accurate timing reconstruction by the receiver (especially important for jitter-sensitive applications like Clock).
