HTTP Trigger Server

The HTTP Trigger Server turns your iOS device into a lightweight web server, allowing external devices (like Stream Deck, Companion, or custom scripts) to trigger actions within Control It.

Configuration

Port

Default: 8080. You can change this if it conflicts with other services, but avoid ports under 1024.

Modes

  • Auto: Binds to WiFi IP if available, falls back to Localhost.
  • Local Only: Binds only to 127.0.0.1 (Loopback). Secure, but no external access.
  • Network Only: Forces binding to the external network interface.

API Reference

The server does not have a fixed API. Instead, you define Custom Endpoints (Triggers) mapped to specific actions.

Request Format

GET /<your-custom-endpoint>

You can also use POST, PUT, DELETE, etc., as configured in the trigger.

Responses

  • 200 OK: Action found and triggered successfully.
  • 404 Not Found: No trigger matches the requested method and path.
  • 401 Unauthorized: Authentication failed (if enabled).

Security

  • Authentication: You can enable Bearer Token authentication for individual triggers. Requests must include the header: Authorization: Bearer <token>.
  • Local Network Only: The server is not accessible from the internet unless you manually port-forward your router (NOT RECOMMENDED).
  • Background Access: iOS will suspend the server when the app is in the background unless "Background Audio" is enabled in settings.
