Control-It Documentation

A powerful iOS app for sending network messages using HTTP, TCP, UDP, OSC, and MIDI protocols with a modular action system, trigger automation, and extensive customization options.

Overview

Control-It is an iOS application designed for professionals who need to send network messages quickly and reliably across multiple protocols. Whether you're controlling lighting systems, audio equipment, video switchers, or any network-enabled device, Control-It provides a unified interface for all your control needs.

Troubleshooting

HTTP Actions Not Working

Symptom: Action fires but nothing happens on target device.

Debug Steps:

  1. Check Connection: Settings ‚Üí Connections ‚Üí Verify IP and port
  2. Review Logs: Settings ‚Üí HTTP Request Logs ‚Üí Check response codes
  3. Test Manually: curl -v http://YOUR_IP:PORT/your/endpoint

Common Fixes:

  • Ensure endpoint starts with /
  • Add Content-Type: application/json header for POST with JSON body
  • Check device firewall allows incoming connections

Error Code Reference

Code Meaning Solution
401 Unauthorized Add authentication header
403 Forbidden Check API permissions
404 Not Found Verify endpoint path
500 Server Error Check server logs
Timeout No response Check network connectivity

MIDI Not Sending

Debug Steps:

  1. Check Destination: Ensure correct MIDI output is selected
  2. Verify Channel: MIDI channels are 1-16 in UI, 0-15 internally
  3. Reconnect After Background: Call midiManager.reconnectAllSources()
  4. Test with MIDI Monitor: Use a MIDI monitor app to verify output

TCP Connection Failing

Debug Steps:

  1. Test Connectivity: ping YOUR_DEVICE_IP
  2. Check Port: nc -zv YOUR_IP PORT
  3. Firewall: Ensure port is open on target device
  4. Try Ephemeral First: Persistent connections may fail if device doesn't support keep-alive

HTTP Trigger Server Issues

Server Won't Start:

  • Ensure WiFi is connected (for network mode)
  • Check if port 8080 is already in use
  • Try Local Only mode for testing

Triggers Not Firing:

  • Verify endpoint path matches exactly (case-sensitive)
  • Check authentication token if enabled
  • Review request in server logs

FAQ

How many actions can I create?
Unlimited actions on all tiers.
How many grids can I have?
Free tier: 5 grids. Pro tier: Unlimited.
Does it work offline?
Yes, for local device control. Cloud features require internet.
What ports does Control-It use?
Outgoing: varies by target device. HTTP Trigger Server: 8080 (configurable).
Can I use it on cellular data?
Yes, with Local Only server mode for same-device triggers.
Which MIDI interfaces work?
USB MIDI (via Camera Connection Kit), Network MIDI, and Bluetooth MIDI.
Can triggers run when app is closed?
HTTP and MIDI triggers require the app to be running (foreground or background with audio).
